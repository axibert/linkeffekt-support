1 · What is LinkEffekt?

LinkEffekt streams audio between music apps and devices — over your local network (via Ableton Link Audio) or over the internet (Online mode). It ships as three AUv3 plugins that run inside any AUv3 host (AUM, Logic Pro, Cubasis, Drambo, GarageBand and others):

PluginTypeUse it to
LinkEffekt SendAudio effectStream a track's audio out — insert it on the track you want to share.
LinkEffekt ReceiveAudio effectReceive channels — works in every host. The standard receiver.
LinkEffekt Receive InstrumentInstrumentThe same receiver as a sound source — for hosts that need one on the channel (e.g. as the source in AUM) or for Logic multi-output routing.

Free vs. Unlimited Channels: the free version includes 1 Send and 1 Receive channel, fully functional on the local network. The one-time in-app purchase Unlimited Channels unlocks unlimited instances, monitoring multiple channels at once, and the Online (internet) mode. Purchase or restore it in the LinkEffekt app or directly inside the plugins.

First step on iOS: open the LinkEffekt app once after installing. This triggers the Local Network permission — without it, the plugins cannot see each other.

2 · Quick start — local network (LAN)

You need two separate apps or devices: Send and Receive share one network identity inside the same host app, so they cannot connect to each other within a single host. Two devices, or two host apps on one device (e.g. Cubasis sends, AUM receives), both work.

  1. Sender: insert LinkEffekt Send on a track. Tap the gear and enable Ableton Link and Share Audio. The orange banner disappears when sharing is active.
  2. Receiver: load LinkEffekt Receive on a track in the other app/device (same Wi-Fi network).
  3. In the receiver, open the Now Receiving menu and pick the sender's channel.
  4. You should hear audio and see the meter move. The status line reads:
Receiving audio.

Tip: name your Send channels (the Name field) — that name is what receivers see. Each additional Send instance appears as its own channel.

3 · Quick start — Online (internet)

Online mode streams over the internet, peer-to-peer, encrypted end-to-end (DTLS). It requires Unlimited Channels on both sides.

  1. Person A: in LinkEffekt Send, switch Destination to Online, tap Create Room and share the room code (copy button next to the code).
  2. Person B: in LinkEffekt Receive, switch the top selector to Online, enter the code and tap Join.
  3. B picks A's channel from Now Receiving. For two-way audio, B also loads a Send (or uses Send my track, see §6).
  4. With Beat Sync on (default), press play on your host transport on both sides and set both hosts to the same tempo — the peer's audio locks onto your beat grid, delayed by whole bars (see §7).

The header shows an ONLINE badge with the room code whenever an instance is in Online mode: green dot = peer connected, orange = waiting for a peer.

4 · LinkEffekt Send — reference

  • Name — the channel name receivers see. Renames propagate live, also in Online rooms.
  • Send Audio — master switch for streaming this track. Off = the plugin passes audio through locally but sends nothing.
  • Mute Local Output (default on) — mutes the track locally after streaming, so you don't hear the signal twice (once dry, once through the receiver with latency). Turn it off only if the receiver is somewhere you can't hear.
  • Gain / Pan — applied to the stream (and to the local output), with click-free ramps. −∞ at the bottom of the gain range.
  • Level meter — post gain/pan; red = clipping (tap to reset).
  • Destination: LAN / Online — where this track streams. Switching Online → LAN removes only this track from the room; the room itself stays open until you tap Leave (a "room is still open" line with a Leave button stays visible in LAN mode).
  • Gear (Ableton Link settings) — the Link session, Share Audio (required for LAN sending; Online is independent of it) and the Link Offset slider (±100 ms) to fine-align audio that arrives consistently early or late. Negative offsets can cause dropouts on the receiver — raise the receiver's latency if needed.
  • Link Audio: Enabled/Disabled — status line, mirrors the Share Audio toggle.

Freemium: the second and further Send instances without Unlimited Channels keep passing audio through locally but do not stream ("Channel Limited" banner with an unlock button).

5 · LinkEffekt Receive — reference

  • LAN / Online selector — which world the channel list below shows. The header shows Link peers (LAN) or the room status (ONLINE badge).
  • Now Receiving — pick exactly one channel. "Channel (unavailable)" means the sender disappeared (app closed, network lost).
  • Monitor multiple channels (Unlimited Channels) — sum several channels into this plugin's stereo output. Channels that are active but belong to the other transport tab are shown with a LAN/Online tag so nothing is summed invisibly. For separate faders per channel, use one Receive instance per channel instead.
  • Network Latency (3–200 ms, default 21.3) — how much audio is buffered. Lower = tighter, higher = safer. If the network needs more, LinkEffekt auto-raises the effective value (shown as "Current: … (auto-adjusted)"); it never lowers your setting on its own.
  • Beat Sync status — "In Sync" (green metronome): the received audio is locked to your beat grid. "Free-run": no shared beat reference; audio plays with minimal latency but is not beat-aligned.
  • Mixer block — in signal order: Received (mute the incoming stream), Host track (Receive effect only; the track's own input — default muted to prevent feedback, see §8), and Output with master gain/pan/mute.
  • Diagnostics line — plain-language status under the channel menu (see §8 for what each message means).
  • Gear → Settings — Ableton Link settings plus Live Mode: reports zero latency to the host so it skips delay compensation. For live playing/jamming only — tracks are not phase-aligned for recording.
  • Packet statistics (gaps/duplicates) appear automatically only when the network has problems.

Receive vs. Receive Instrument: identical engine. Use the effect everywhere you can insert it; use the Instrument where the channel needs a source (AUM source slot, Logic software-instrument track with multi-output).

6 · Online mode in detail

  • One room per host app. All Online instances inside the same host share one room and one connection. The first instance creates/joins; the others follow automatically. Leave disconnects all of the host's online tracks (the dialog says so).
  • Multiple tracks: every Send-Online instance is its own stream — the peer sees each one as a separate channel (up to 8 per direction).
  • Send my track (Receive effect in Online mode) — also streams this track's input into the room: full two-way audio in a single plugin slot. Built for hosts with only one usable slot, e.g. DJ apps.
  • Room codes are one-time invitations relayed by the signaling server; the audio itself flows peer-to-peer, DTLS-encrypted. No audio passes through our servers. Rooms with only one participant close after 30 minutes; active sessions have no time limit.
  • Reconnect: if one side drops out, rejoin with the same code while the other side is still in the room. If both left, create a fresh room.
  • Mono send (optional, in the Send settings) halves the bandwidth — useful on weak connections.

7 · Beat Sync — how it works and what to expect

Audio that travels through a network always arrives late; no software can remove that. What LinkEffekt can do is make the delay musical:

  • LAN: the receiver places the incoming audio onto your local Ableton Link grid. The result is a stable, grid-locked offset roughly equal to the Latency setting — instead of drifting timing. Both apps must be in the same Link session (Link enabled on both).
  • Online: the delay is rounded up to whole bars on your host's beat grid — the peer's "1" lands exactly on your "1", typically one bar later (2 s at 120 BPM). Requirements: Beat Sync on (default), host transport playing on both sides, same tempo on both hosts. When you press play, expect about one bar of silence while the buffer fills, then it locks.
  • Free-run is what you get without a shared beat reference: lowest latency, no alignment — fine for monitoring, speech, or when you play by ear.
  • Recording into Ableton Live (LAN): enable Sync to Incoming Audio under Live's Settings → Link for sample-accurate placement; set Live's Link-Audio latency slider generously (100–200 ms) and Monitor to In.

The two hosts' own metronomes are not aligned to each other over the internet — each side hears the peer on their own grid. In a remote jam that is exactly what you want, since each player only hears their own device.

8 · Troubleshooting

No channel appears in Now Receiving (LAN)

  • Send and Receive must run in different apps or devices (§2).
  • Same Wi-Fi network? Some routers isolate clients ("AP/client isolation") — disable that, or use a hotspot.
  • Open the LinkEffekt app once (Local Network permission). The plugin shows an orange banner if this is missing.
  • On the sender: gear → Share Audio must be on.

Channel selected, but no sound — read the diagnostics line

No audio from the sender… — the sender isn't streaming: Share Audio off, Send Audio off, or a different network.
Receiving data, but it is silent… — connection is fine; the sender's track has no signal.
…the host is not running this plugin. — the host never renders the channel. In AUM: give the channel a source, or use Receive Instrument as the source.
Syncing to the beat — buffering… — normal for a moment; if it persists, raise the Latency slider.
Receiving audio, but "Mute Received" / "Mute Output" is ON — un-mute the control it names (also shown when the master gain sits at −∞).
Free limit reached… — second Receive instance without Unlimited Channels; unlock or remove the other instance.
No audio from the peer yet… — (Online) the peer left the room or isn't streaming.

Crackles or dropouts

Raise the Network Latency slider until clean, then add a small margin. On weak Wi-Fi move closer to the router; in Online mode try Mono send.

Feedback howling

The receiving track has a live mic input feeding through. Keep Host track muted (the default) in the Receive mixer, or use the Receive Instrument (no input at all).

I hear the track twice (sender side)

That is the dry signal plus the stream. Keep Mute Local Output on in the Send.

Online: "Room not found"

Code typo (codes are letters/numbers only; pasted spaces are removed automatically), or the room expired because it stayed empty. Create a fresh room.

Online: stuck on "waiting for peer"

The other side hasn't joined yet or joined a mistyped code. Re-share the code; both sides need Unlimited Channels.

Beat Sync won't lock (Online)

Is the host transport playing on both sides? Same tempo? The status line names what's missing — a tempo mismatch is shown as "you X / peer Y BPM".

Timing is stable but consistently early or late (LAN)

Use the Link Offset slider in the Send's gear settings to trim ±100 ms.

9 · Frequently asked questions

Does LinkEffekt work with Ableton Live? Yes — Live 12.4+ receives Link Audio directly (§7). LinkEffekt Send appears there as an input channel.

How many channels can I run? With Unlimited Channels: as many Send/Receive instances as your device handles; Online supports up to 8 streams per direction per host.

Does audio go through your servers? No. Online audio is peer-to-peer and DTLS-encrypted; the server only relays the room handshake. If a direct connection is impossible, an encrypted relay (TURN) forwards the still-encrypted stream.

Why is there always some delay? Physics: network transit plus safety buffering. LinkEffekt makes the delay stable and — with Beat Sync — musical (§7).

Which sample rates are supported? The host's rate (44.1/48/96 kHz…) on both sides; LinkEffekt converts internally. Online always transmits at 48 kHz/Opus.
